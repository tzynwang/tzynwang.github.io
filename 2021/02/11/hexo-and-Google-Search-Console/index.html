提問：如何讓Google Search可以搜尋到部屬在GitHub Page上的hexo部落格？
簡答：去Google Search Console對Google Search註冊你的hexo部落格

大部分解🔫內收

本篇用語說明

  • hexo blog：意指Setup時，透過hexo init <folder>產生，存放hexo內容的資料夾
  • cmd.exe：Windows內建的命令列直譯器，按住鍵盤上的Windos鍵（通常在鍵盤左下角）加上R鍵，輸入cmd後Enter即可開啟cmd.exe
  • GitHub Page：指已經完成hexo deploy，被部屬到GitHub Page的hexo部落格；可以透過「https://<github使用者名稱>.github.io」這個網址瀏覽。

步驟

  1. 開啟Google Search Console
  2. 在「網頁前置字元」輸入GitHub Page的網址
  3. 選擇「其他驗證方法」中的「HTML 標記」，複製由Google提供的中繼標記
  4. 進入hexo部落格在本機的資料夾
  5. 進入themes資料夾→（主題名稱資料夾）→layout→_partial
  6. 開啟head檔案，將前一個步驟複製起來的Google中繼標記貼到標籤之中
  7. 開啟cmd.exe，移動到hexo blog（cd <hexo blog的路徑>），執行hexo generate --deploy
  8. 使用瀏覽器Chrome或Brave開啟「https://<github使用者名稱>.github.io」，按下F12呼叫開發者工具，確認之間有包含Google中繼標記
  9. 回到Google Search Console，點選驗證；驗證成功即代表網站註冊完成

筆記

提問：為什麼要把中繼標記貼到themes資料夾裡面的head檔案中？
答：每次執行hexo generate時，才會確保中繼標記都有被包進最後要部屬上GitHub的檔案裡面

提問：沒有把中繼標記的內容存下來，哪裡可以找回來？
答：

  1. 進入Google Search Console，左側欄位往下捲，點選「設定」
  2. 右側面板點選「使用者和權限」，點選擁有者右側的三點按鈕，點選「管理資源擁有者」，會另開新視窗
  3. 點選新視窗中的「詳細資料」複製中繼標記
